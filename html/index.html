Introduction

Cette API transforme les données officielles de la Base de Données Publique des Médicaments (BDPM) en une interface RESTful optimisée, en utilisant un parsing concurrent de 5 fichiers TSV (spécialités, compositions, présentations, génériques, conditions) avec validation et structuration automatique.


Idéale pour les développeurs, chercheurs, et professionnels de santé qui ont besoin d'accéder programmatiquement aux informations sur les médicaments disponibles en France.


🚀 Quick Start

curl https://medicamentsapi.giygas.dev/medicament/paracetamol
curl https://medicamentsapi.giygas.dev/database/1
curl https://medicamentsapi.giygas.dev/health

Fonctionnalités Principales

🔍

Recherche avancée

Recherchez des médicaments par dénomination (regex insensible à la casse) ou par code CIS unique avec validation d'entrée stricte (3-50 caractères alphanumériques). Performance O(1) via lookup maps.

GET /medicament/paracetamol
📄

Pagination optimisée

Parcourez la base complète avec pagination optimisée (10 éléments/page) et métadonnées complètes (totalItems, maxPage, currentPage). Cache HTTP intelligent pour performances maximales.

GET /database/{pageNumber}
🔄

Zero-downtime updates

Synchronisation automatique à 6h et 18h avec atomic operations pour zero-downtime updates. Monitoring de santé intégré avec alertes si >25h sans mise à jour.

Atomic swap <2s
⚡

Cache HTTP intelligent

Réponses optimisées avec cache HTTP (ETag, Last-Modified), compression gzip (80% réduction) et support 304 Not Modified. Timeouts configurables (15s read/write).

Cache hit ratio >90%
🔒

Rate limiting intelligent

Protection par token bucket (1000 tokens initiaux, recharge de 3 tokens/sec) avec coûts variables par endpoint (5-200 tokens). Nettoyage automatique des clients inactifs.

X-RateLimit-* headers
📊

Données BDPM complètes

Accès complet aux données BDPM : compositions, présentations, génériques, conditions, voies d'administration et statuts AMM. 20K+ médicaments avec 5 sources TSV parsées concurrently.

20MB dataset ~50MB RAM

Points de terminaison

Endpoint Description Cache Coût
GET /database Base de données complète 6h 200
GET /database/{page} Pagination (10 éléments/page) 6h 20
GET /medicament/{nom} Recherche par nom (regex) 1h 100
GET /medicament/id/{cis} Recherche par identifiant CIS 12h 100
GET /generiques/{libelle} Génériques par libellé 1h 20
GET /generiques/group/{id} Groupe générique par ID 12h 20
GET /health État de santé du service - 5
GET /docs Documentation interactive 1h 0

Exemples d'utilisation

Découvrez comment utiliser l'API avec ces exemples pratiques dans différents langages.

📡 Recherche de médicaments

cURL 
curl "https://medicamentsapi.giygas.dev/medicament/paracetamol" \
  -H "Accept-Encoding: gzip" \
  -H "Accept: application/json"
JavaScript 
const response = await fetch('https://medicamentsapi.giygas.dev/medicament/paracetamol');
const data = await response.json();
console.log(`Found ${data.count} medicaments`);
Python 
import requests
response = requests.get('https://medicamentsapi.giygas.dev/medicament/paracetamol')
data = response.json()
print(f"Found {data['count']} results")

📊 Pagination

curl "https://medicamentsapi.giygas.dev/database/1"
# Response:
{
  "data": [...],
  "page": 1,
  "pageSize": 10,
  "totalItems": 15420,
  "maxPage": 1542
}

💊 Génériques

Recherche de génériques 
curl "https://medicamentsapi.giygas.dev/generiques/paracetamol"
Groupe générique par ID 
curl "https://medicamentsapi.giygas.dev/generiques/group/1234"

🔒 Rate Limiting

# Vérifier les headers de rate limiting
curl -I "https://medicamentsapi.giygas.dev/health"

# Headers retournés:
X-RateLimit-Limit: 1000      # Capacité maximale
X-RateLimit-Remaining: 850   # Tokens restants  
X-RateLimit-Rate: 3          # Taux de recharge (tokens/sec)
